How to Set Up Django - Step by Step

A beginner-friendly guide to setting up Django for your first project.

Step 1: Install Python and Django

Install Python (if not already installed):

# On macOS/Linux:
# brew install python

# On Windows, download the installer from https://www.python.org/

Install Django using pip:

pip install django

Step 2: Create a Django Project

Run the following command to create your first Django project:

django-admin startproject myproject

This will create a folder named myproject containing the necessary files.

Step 3: Run the Development Server

Navigate to your project folder and start the server:

cd myproject
python manage.py runserver

Visit http://127.0.0.1:8000/ to see the Django welcome page!

Step 4: Create a Django App

Create a new app by running the following command:

python manage.py startapp myapp

A new folder named myapp will be created.

Step 5: Configure Your App

Open myproject/settings.py and add 'myapp' to the INSTALLED_APPS list:

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'myapp',
]

Step 6: Define and Include URL Patterns

Create a URL Configuration File: In your app directory myapp, create a file named urls.py (if it doesn’t already exist).

# myapp/urls.py
        from django.urls import path
        from . import views  # Import views from your app
        
        urlpatterns = [
            path('', views.index, name='index'),  # Define a URL pattern
        ]

This defines a URL pattern that maps the root URL of your app (/) to the index view function.

Modify the Project’s Main URL Configuration: In your project folder, open myproject/urls.py and include your app's URL patterns:

# myproject/urls.py
        from django.contrib import admin
        from django.urls import path, include  # Import include
        
        urlpatterns = [
            path('admin/', admin.site.urls),
            path('', include('myapp.urls')),  # Include app URLs
        ]

The include() function allows you to reference the app’s urls.py in the main URL configuration.

Create the HTML File: In your app directory myapp, create a folder named templates. Inside it, create another folder named myapp (this ensures proper template loading). Then create a file index.html inside templates/myapp:

<!DOCTYPE html>
        <html lang="en">
        <head>
            <meta charset="UTF-8">
            <meta name="viewport" content="width=device-width, initial-scale=1.0">
            <title>Welcome to Django</title>
        </head>
        <body>
            <h1>Hello, Django!</h1>
            <p>This is your first rendered HTML page.</p>
        </body>
        </html>

Update the View: Open myapp/views.py and update the index view function to render the HTML file:

# myapp/views.py
        from django.shortcuts import render  # Import render function
        
        def index(request):
            return render(request, 'myapp/index.html')  # Render the HTML template

This function uses the render method to load and display the index.html template.

Test Your Setup: Start the development server:

python manage.py runserver

Visit http://127.0.0.1:8000/ in your browser. You should see the content of index.html rendered as a webpage!

Challenges Faced While Building This Project

Understanding Django's Structure: While Django is straightforward to set up and run, adapting to its framework structure took some time. Django's approach to organizing projects and apps, while efficient, can initially feel unfamiliar.

Working with Multiple Apps: One of Django's strengths is its ability to handle multiple apps within a single project. However, managing the separation of functionality between these apps and understanding their individual roles posed a slight learning curve at the start.

Flexibility in URL Naming: Defining custom URL route names was both a helpful and challenging feature. While it provides flexibility, ensuring consistency and readability across routes required careful planning.

Rendering Templates: This was the most challenging part for me. Django enforces specific naming conventions for creating templates and organizing folder structures, which initially caused confusion. Understanding how to properly set up the templates directory and correctly link HTML files required extra effort but proved rewarding in the end.
